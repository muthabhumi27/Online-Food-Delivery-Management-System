Q1
Top 3 Restaurants by Average Rating & Revenue
Finds the best restaurants using multi-table JOINs, averaging ratings and summing revenue. Only restaurants with more than 5 orders qualify (HAVING clause).
JOIN AVG · SUM HAVING COUNT > 5 
SELECT r.RestaurantName,
       AVG(rev.Rating)              AS AvgRating,
       SUM(oi.Quantity * oi.ItemPrice) AS TotalRevenue
FROM Restaurant r
JOIN Review   rev ON r.RestaurantID = rev.RestaurantID
JOIN `Order`  o   ON r.RestaurantID = o.RestaurantID
JOIN OrderItem oi  ON o.OrderID      = oi.OrderID
GROUP BY r.RestaurantID, r.RestaurantName
HAVING COUNT(o.OrderID) > 5
ORDER BY AvgRating DESC, TotalRevenue DESC
LIMIT 3;

Q3
Orders With Unavailable Items & No Payment
Detects problematic orders: items marked unavailable in the menu, AND no successful payment on record. Uses NOT EXISTS for the payment check.
NOT EXISTS JOIN · WHERE 
SELECT o.OrderID, c.CustomerName
FROM `Order` o
JOIN Customer  c  ON o.CustomerID  = c.CustomerID
JOIN OrderItem oi ON o.OrderID     = oi.OrderID
JOIN FoodItem  fi ON oi.FoodItemID = fi.FoodItemID
WHERE fi.Availability = FALSE
  AND NOT EXISTS (          -- no paid payment for this order
      SELECT 1
      FROM Payment p
      WHERE p.OrderID = o.OrderID
        AND p.PaymentStatus = 'Paid'
  );

Customer Order Summary View
Queries the CustomerOrderSummary VIEW (pre-created in DB) which aggregates total orders and spend per customer. Filters customers spending over ₹100.
CREATE VIEW SUM · COUNT 
-- Step 1: VIEW definition (already created in setup.sql)
CREATE VIEW CustomerOrderSummary AS
SELECT c.CustomerID, c.CustomerName,
       COUNT(o.OrderID)              AS TotalOrders,
       SUM(oi.Quantity * oi.ItemPrice) AS TotalSpent
FROM Customer c
JOIN `Order`   o  ON c.CustomerID = o.CustomerID
JOIN OrderItem oi ON o.OrderID    = oi.OrderID
GROUP BY c.CustomerID, c.CustomerName;
 
-- Step 2: Query the view like a regular table
SELECT * FROM CustomerOrderSummary

ORDER BY TotalSpent DESC;

Top Delivery Partners — Last Month
Finds the most productive delivery partners by counting completed deliveries within the last 30 days using DATE_SUB(CURDATE(), INTERVAL 1 MONTH).
DATE_SUB · CURDATE COUNT · GROUP BY 
SELECT dm.Name AS DeliveryManName,
       COUNT(o.OrderID) AS Deliveries
FROM DeliveryPartner dm
JOIN `Order` o ON dm.OrderID = o.OrderID
WHERE o.OrderStatus = 'Delivered'
  AND o.OrderDate >= DATE_SUB(CURDATE(), INTERVAL 1 MONTH)
GROUP BY dm.PartnerID, dm.Name
ORDER BY Deliveries DESC
